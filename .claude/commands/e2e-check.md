# E2E Check

핵심 사용자 플로우를 Playwright MCP로 자동 검증하고 성공/실패 리포트를 생성한다.

## 사용법

```
/e2e-check [flow]
```

`flow` 생략 시 전체 플로우를 순서대로 실행한다.

| flow 인자 | 검증 내용 |
|-----------|----------|
| `health` | 헬스 체크 엔드포인트 응답 확인 |
| `home` | 메인 페이지 로딩 + JS 에러 0건 확인 |
| `agent` | 에이전트 호출 (입력 → 응답) 플로우 |
| `auth` | 로그인 / 로그아웃 (구현된 경우) |

예시:
- `/e2e-check` — 전체 플로우
- `/e2e-check agent` — 에이전트 플로우만

## 전제조건

- Playwright MCP 활성화 (`.mcp.json` 참조)
- 백엔드 dev server: `cd backend && npm run dev`
- (선택) 정적 프론트는 백엔드가 함께 서빙하거나 별도 서버 실행
- 테스트 시나리오는 `qa/` 디렉토리의 Playwright 스크립트도 참조

## 플로우 정의

### Flow 1: health — 헬스 체크

```
1. browser_navigate("http://localhost:3100/health")
2. 응답 본문에 status 필드 확인
3. 콘솔 에러 0건 확인
```

검증 포인트:
- [ ] 200 응답
- [ ] JSON 본문 파싱 가능
- [ ] 콘솔 에러 0건

### Flow 2: home — 메인 페이지

```
1. browser_navigate("http://localhost:3100/")
2. 페이지 스냅샷 → 주요 영역 표시 확인
3. 콘솔 에러 0건 확인
4. 네트워크 4xx/5xx 0건 확인
```

검증 포인트:
- [ ] 메인 컨텐츠 영역 렌더링
- [ ] 정적 리소스 200 응답
- [ ] 콘솔 에러 없음

### Flow 3: agent — 에이전트 호출

```
1. /agents 또는 메인 입력 폼 이동
2. 입력란에 "테스트 프롬프트" 입력
3. 실행/전송 버튼 클릭
4. 응답 영역 갱신 대기
5. 응답 텍스트가 비어 있지 않음 확인
6. 백엔드 API 호출 200 응답 확인
```

검증 포인트:
- [ ] 입력 폼 동작
- [ ] 응답 영역 업데이트
- [ ] 백엔드 응답 정상

### Flow 4: auth — 인증 (구현 시)

```
1. /login 이동
2. 자격증명 입력 → 로그인
3. 로그인 후 페이지 전환 확인
4. 로그아웃 버튼 → 로그인 페이지 복귀 확인
```

검증 포인트:
- [ ] 로그인 성공 후 보호 경로 접근 가능
- [ ] 로그아웃 후 보호 경로 차단

## 오류 처리 및 디버깅

각 단계 실패 시:

1. **스크린샷 캡처** — 현재 화면 상태 저장
2. **콘솔 로그 확인** — `browser_console_messages` 로 JS 에러 확인
3. **네트워크 요청 확인** — API 응답 상태 코드 확인
4. **원인 분류**:
   - 프론트엔드 렌더링 오류 → `frontend/` 코드 확인
   - API 에러 (4xx/5xx) → `backend/` 로그 확인
   - 인증 오류 → 토큰/쿠키 상태 확인
